# ホテル顧客リクエスト管理システム (Hotel Request API)

ホテルスタッフとして働いていた際に作った「スタッフ間の情報共有の漏れをなくす」ことを目的とした管理ツールをNode.jsを用いて開発しました。

##  開発の背景と目的
ホテルの現場では、お客様からの「タオルを追加してほしい」「チェックアウトを延長したい」といったリクエストが絶えません。
これらを口頭やメモだけで管理するとミスが起きやすいため、リアルタイムで進捗（未対応・対応中・完了）を可視化できるシステムを目指して作成しました。

##  実装した主要機能
- **ステータス管理:** 各リクエストの対応状況をワンクリックで更新。

- **スクロールUI:** 現場での使い勝手を考慮し、一覧性が高いデザインを採用。
- **REST API:** Node.js (Express) を使用し、データの取得や更新をスムーズに行える設計。

##  Node.js (Backend) の役割
本システムにおいて、Node.jsは「情報の交通整理」を行う重要な役割を担っています。

- **APIサーバーの構築:** フロントエンド（画面）からの「ステータスを変えてほしい」「顧客を検索したい」というリクエストを、Expressフレームワークを用いて適切に処理しています。
- **データ通信の制御:** 画面をリロードすることなく、必要なデータだけをやり取りすることで、ストレスのない操作感を実現しています。
- **ビジネスロジックの集約:** フロントエンドから届いたリクエストを解析し、適切なレスポンスを返すためのバックエンド処理をJSで一貫して記述しています。

##  使用技術
- **Backend:** Node.js / Express
- **Frontend:** HTML5 / CSS3 / JavaScript (Vanilla JS)
- **Tool:** VS Code / Git / GitHub


##  デザインのこだわり
実際のホテル業務での利用シーンを想定し、以下の点にこだわって設計しました。

- **視認性の高いステータス管理:** 対応状況を「未対応（赤）」「対応中（黄）」「完了（青）」のように色分けし、一目で優先順位が判断できるようにしました。
- <img width="1204" height="769" alt="スクリーンショット 2026-04-14 221133" src="https://github.com/user-attachments/assets/cd7a7a3f-c801-4792-b18d-9f98c2902ff0" />

- **直感的な操作感:** 現場の忙しさを考慮し、マウスの移動距離を最小限に抑えたボタン配置と、迷わないシンプルなレイアウトを採用しています。
- **情報の整理:** リクエスト内容、顧客名、をカード形式で整理し、狭いタブレット端末などの画面でも情報が重ならないよう調整しました。
